# Forest and Typescript

Because Forest is a database of dynamic values, it is not *directly* typeable; there are too many internal variations to
conveniently type-lock the core Forest class itself. Similarly, actions are user-defined, so you cannot define the type
of arguments in the definition of a forest's action, as exposed on the 'do' property.

However, you can put type "around" the value; as in:

````typescript
import type { leafI } from "@wonderlandlabs/forest/lib/types";
import {Forest} from "@wonderlandlabs/forest";
import {useState, useEffect} from 'react';

type PointValue = {
  x: number,
  y: number,
  label: string
}

interface PointValueInterface extends leafI {
  value: PointValue,
}

const initial: PointValue = {
  x: 0,
  y: 0,
  label: 'Home'
}

const pointValue = new Forest({
  $value: initial,
  actions: {
    moveX(leaf: leafI, offset: number) {
      leaf.do.set_x(leaf.value.x + offset);
    }

  }
}) as PointValueInterface

const [value, setValue] = useState<PointValue>(pointValue.value);
useEffect(() => {
  const sub = pointValue.subscribe(setValue);
  pointValue.do.loadData();
  pointValue.do.loadCountries();

  return () => sub.unsubscribe();
}, [pointValue]);

````
